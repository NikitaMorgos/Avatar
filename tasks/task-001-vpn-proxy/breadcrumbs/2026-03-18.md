# 2026-03-18 — Диагностика и старт задачи

## Проблема
Collect bot периодически падал с ошибками:
```
Failed to set commands: HTTP Client says - Request timeout error
Cannot connect to host api.telegram.org:443 ssl:default [None]
TelegramNetworkError: Request timeout error
```
Причина: блокировка `api.telegram.org` в РФ без VPN.

## Лог последнего падения
```
2026-03-17 21:18:23 - Polling started
2026-03-17 21:19:23 - Polling stopped
TelegramNetworkError: Request timeout error (при bot.get_me())
exit_code: 1, elapsed_ms: 193054
```

## Сделано
- Добавлена поддержка прокси в `bot/collect_bot.py`:
  - импорт `AiohttpSession`
  - переменная `PROXY` из `.env`
  - `Bot(token=BOT_TOKEN, session=AiohttpSession(proxy=PROXY))`
- `aiohttp-socks>=0.8.0` добавлен в `requirements.txt`
- `.env`: добавлены закомментированные примеры PROXY

## Решение
VPS на Timeweb Cloud (Германия, Франкфурт) + SSH SOCKS5-туннель.
Ждём IP VPS от пользователя.

## Альтернативы (рассматривались)
- Использовать имеющийся VPN-клиент — отклонено (нестабильно, зависит от сторонних сервисов)
- Outline — отклонено (не даёт локальный SOCKS5 для программного использования)
- Собственный WireGuard — отклонено (маршрутизирует весь трафик, сложнее)
