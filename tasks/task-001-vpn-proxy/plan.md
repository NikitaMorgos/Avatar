# task-001-vpn-proxy — План

## Цель
Обеспечить стабильную работу Collect bot в условиях блокировок Telegram в РФ.
Решение: собственный VPS на немецком сервере Timeweb Cloud + SSH SOCKS5-туннель.

## Подход
SSH SOCKS5-туннель: `ssh -D 1080 root@VPS_IP` → локальный прокси → aiogram `AiohttpSession(proxy=...)`

## Шаги

### Фаза 1 — Инфраструктура (требует действий пользователя)
- [ ] Зарегистрироваться / войти на [timeweb.cloud](https://timeweb.cloud)
- [ ] Создать VPS: Германия (Франкфурт), Ubuntu 22.04, минимальный тариф (~660 руб/мес)
- [ ] Получить IP-адрес VPS
- [ ] Сгенерировать SSH-ключ (если нет) и добавить в Timeweb
- [ ] Проверить доступ: `ssh root@VPS_IP`

### Фаза 2 — Автотуннель (готов к реализации)
- [x] Код прокси добавлен в `bot/collect_bot.py` (`AiohttpSession(proxy=PROXY)`)
- [x] `aiohttp-socks` добавлен в `requirements.txt`
- [x] Переменная `PROXY` добавлена в `.env` (закомментирована)
- [ ] Создать `deploy/start-tunnel.ps1` — автозапуск SSH-туннеля с reconnect
- [ ] Создать `deploy/start-avatar.ps1` — единый старт: туннель + бот
- [ ] Установить `aiohttp-socks`: `pip install aiohttp-socks`

### Фаза 3 — Активация
- [ ] Вставить IP VPS в скрипты и `.env`
- [ ] Раскомментировать `PROXY=socks5://127.0.0.1:1080` в `.env`
- [ ] Запустить `deploy/start-avatar.ps1`
- [ ] Проверить: бот отвечает на фото в Telegram

### Фаза 4 — Надёжность (опционально)
- [ ] Добавить задачу в Планировщик задач Windows (автостарт при входе)
- [ ] Настроить мониторинг: алерт если бот не отвечает >5 мин
