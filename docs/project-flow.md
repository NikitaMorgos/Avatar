# Проект Avatar: флоу и роль Perplexity

## Кто за что отвечает

- **Perplexity** — проект‑менеджер и архитектор. Читает документацию, предлагает структуры, формирует ТЗ.
- **Cursor** — инструмент разработки. Реализует код по документации и ТЗ.
- **Ты** — связующее звено: формулируешь цели, передаёшь ТЗ от Perplexity к Cursor.

---

## Где Perplexity берёт контекст

Он читает:

1. `docs/avatar-project.md` — концепт Avatar, области жизни, карта модулей
2. `docs/collect-spec.md` — спецификация модуля Collect
3. `docs/project-flow.md` — этот файл
4. `README.md` — что это за проект, как запускать

Всё это — обычные markdown‑файлы. Их можно скопировать в Perplexity целиком или частями и просить: «Предложи структуру для следующей итерации» или «Разбей цель X на задачи».

---

## Как планируем итерации

1. **Ты формулируешь цель** — например: «Нужно добавить команду /export для Collect».
2. **Perplexity** — разбивает на задачи, дополняет документацию (например, обновляет `collect-spec.md`), формирует ТЗ.
3. **Ты даёшь ТЗ Cursor** — Cursor реализует код по описанию и существующей структуре проекта.

Без лишней теории: Perplexity — «мозг» планирования, Cursor — «руки» реализации.
