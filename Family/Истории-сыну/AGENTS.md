> Этот файл читается AI-агентами для понимания контекста проекта перед любой работой с материалом.

---

## Что такое этот проект

**Stories for Mironchi** — папа (Никита) рассказывает сыну (Мирончи, 6 лет) истории про своё детство перед сном. Мирончи угадывает каждую историю с одного слова.

Цели проекта:
- **Книга** — литературная обработка историй с авторским голосом
- **ТГ-канал** — `t.me/StoriesForMironchi` — публичные посты
- **Педагогика** — советы, что проговорить с сыном, вопросы, активности
- **Архив** — сохранить 100+ историй в структурированном виде

## Структура папок

```
Family/Истории-сыну/
  AGENTS.md          ← этот файл
  transcribe.py      ← скрипт транскрипции через AssemblyAI
  audio/             ← исходные m4a-записи (добавлять сюда новые)
  transcripts/
    ИМЯ_raw.txt      ← сырой транскрипт из AssemblyAI (не редактировать)
    ИМЯ_clean.txt    ← почищенный транскрипт (убран мусор, оставлены реплики)
  processed/
    ИМЯ.md           ← финальная обработка: текст + педагогика + советы
```

---

## Рабочий поток (на каждую порцию ~10 историй/неделю)

```
1. Никита записывает аудио (Plaud или телефон)
2. Кладёт m4a файл в audio/
3. Запускает транскрипцию: python transcribe.py
4. Сырой транскрипт появляется в transcripts/ИМЯ_raw.txt
5. Присылает тексты агенту (или агент читает сам из папки)
6. Агент создаёт transcripts/ИМЯ_clean.txt и processed/ИМЯ.md
7. Агент обновляет раздел Семья на сайте Avatar
```

## Как чистить транскрипт (→ ИМЯ_clean.txt)

**Убрать:**
- Вступление с угадыванием (игра «первое слово»)
- Технические реплики («подожди, я записываю», «нажимай»)
- Шум, повторы, явные оговорки

**Оставить:**
- Саму историю целиком
- Реплики Мирончи по ходу — это живая фактура, помечать `[сын]:`
- Самоисправления рассказчика — это его голос

## Формат processed/ИМЯ.md

Каждый файл содержит три блока:

### 📖 Текст (авторский голос)

**Принципы:**
- Минимальная литературная обработка — слышен живой рассказчик
- Короткие рубленые фразы — разговорный ритм Никиты
- Диалоги с Мирончи вшиты в текст как живые сцены (не сноска)
- Сына называть **Мирончи** (не Миронка, не Миронкой)
- Прямые цитаты оформлять через тире без кавычек:
  ```
  — Авоська? Объясни, что это такое.
  — Ну это такой... она вешается на два дерева...
  ```
- Финальная реплика Мирончи — хорошая точка для конца истории
- Никаких «литературных» украшений и штампов

**Пример хорошего финала** (из «Крутобол»):
> — А мы же ездили туда год назад.
> — Ну вот, тогда мы его и видели.
> — Точно видели. В прошлом году, значит.
>
> Ладно, конец этой истории.

### 🎓 Педагогическая рамка

- Одно предложение: что за история, в чём её суть
- Список из 2–3 пунктов: что проговорить с сыном (возраст 6 лет)
- Конкретный вопрос для следующего раза (в кавычках)

### 💬 Что делать дальше

- 2–3 конкретных активности / ритуала по мотивам истории
- Вопрос на засыпание

## Фотографии к историям

### Способ 1 — UI на сайте (основной, хранится в localStorage)

В каждой истории под заголовком «📖 История» есть кнопка **📷 Добавить фото**.

**Как использовать:**
1. Открыть раздел Семья → история → нажать «📷 Добавить фото»
2. Выбрать файл с устройства + написать подпись
3. Нажать «Сохранить»

Фото сохраняется как base64 в `localStorage` браузера, подпись — туда же.  
При следующем открытии раздела восстанавливаются автоматически через `sPhotoInit()`.  
Кнопка меняется на «✏️ Изменить фото» — можно обновить фото или подпись в любой момент.

**localStorage ключи:**
- `story_photo_gnomik` — base64 фото истории «Гномик Гоша»
- `story_caption_gnomik` — подпись к фото
- (аналогично для `krutobol`, `ozhog`, `pudra`, `transformator`)

**Ограничение:** хранится только в текущем браузере. При смене браузера/устройства — нужно добавить заново.

---

### Способ 2 — файловый (через git, виден всем)

Если нужно чтобы фото появлялось у всех (не только в своём браузере):

1. Положить фото в `photos/gnomik.jpg`
2. Скопировать в `Avatar/assets/stories/gnomik.jpg`
3. `git add assets/stories/gnomik.jpg && git commit && git push`

Если `assets/stories/XXX.jpg` существует — он показывается. Если загружено через UI — UI-фото (localStorage) перезаписывает файловое в браузере того, кто добавил.
